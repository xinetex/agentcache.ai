# Design Partner Operating Playbook

Status: Active  
Owner: AgentCache.ai  
Date: 2026-03-14

## 1. Objective

Convert the first `3` design partners into measurable proof that AgentCache Memory Fabric saves money, reduces latency, and improves evidence quality.

## 2. Partner Selection Rule

Take a design partner only if all of the following are true:

- they have real or imminent agent workloads
- they can expose baseline metrics
- they have a technical owner available
- the workflow is narrow enough to instrument in `4-6` weeks
- the buyer agrees on success criteria before implementation

Reject or defer if:

- they want broad consulting
- they need months of procurement before technical validation
- they cannot define a target workflow
- they refuse measurement

## 3. Qualification Scorecard

Score each prospect from `1-5` on:

- usage pain
- speed of integration
- willingness to instrument
- strategic fit with enterprise-copilot or finance-memory-fabric
- case-study value

Prioritize prospects scoring `18+`.

## 4. Discovery Call Structure

### Questions to answer

- what workflow repeats enough to benefit from caching or memory reuse?
- what is the current cost or latency pain?
- what retrieval or browser steps are involved?
- what trust or evidence gaps exist today?
- who can integrate and own the rollout?
- what would make this pilot an obvious success?

### Deliverable after call

Produce a one-page pilot definition with:

- workflow name
- current stack
- baseline metrics
- target metrics
- required integration points
- timeline
- named owner on both sides

## 5. Pilot Shape

Every pilot should be a constrained installation.

### In scope

- one workflow
- one SKU
- one sector policy
- one baseline
- one ROI report

### Out of scope

- broad platform replacement
- many unrelated workflows
- large customization without clear reuse
- vague transformation projects

## 6. Implementation Sequence

### Week 1

- define workflow
- collect baseline
- choose SKU
- finalize success metrics

### Week 2

- connect API path
- enable policy-aware cache or proof path
- validate receipt and evidence generation

### Weeks 3-4

- run traffic
- review hit rate, latency, and evidence coverage
- tighten policy and proof settings

### Weeks 5-6

- generate ROI summary
- review savings and trust delta
- convert to paid production plan

## 7. Canonical Success Metrics

Use the same scorecard in every pilot:

- hit rate
- p50 latency reduction
- p95 latency reduction
- estimated cost saved per 1k tasks
- evidence coverage rate
- freshness compliance rate

For finance-oriented pilots, also track:

- audit-ready proof coverage
- stale-read avoidance
- browser-proof completion rate

## 8. Pricing Discipline

Preferred order:

1. paid pilot
2. discounted pilot with strong case-study rights
3. unpaid pilot only if the logo, data quality, and speed justify it

Never do an unpaid pilot without:

- explicit scope
- named owner
- data access for measurement
- defined conversion criteria

## 9. Conversion Rule

A pilot converts to production only if:

- one or more core metrics moved materially
- the partner confirms operational value
- the deployment path is repeatable
- the account has a clear paid expansion path

If the pilot does not move metrics, treat it as a product learning event, not a sales win.

## 10. Case Study Template

Every successful pilot should produce:

- customer/problem summary
- baseline metrics
- implementation scope
- results after AgentCache
- customer quote if available
- next expansion opportunities

Prefer quantified claims over qualitative praise.

## 11. Internal Weekly Review

Review every active prospect and pilot weekly:

- qualification score
- current stage
- blocker
- next action
- owner
- probability of conversion

If a pilot has no clear next action, it is drifting.

## 12. Product Feedback Loop

Push recurring objections directly back into product priorities.

Highest-priority feedback types:

- integration friction
- unclear pricing or packaging
- weak ROI visibility
- missing evidence exports
- sector-policy mismatch

## 13. Relationship to Other Product Lines

- `AgentCache` is the sales front door
- `MaxxEval` supports trust and evidence packaging when useful
- `ClawSave` supplies dogfooding and internal proof only
- `Symbiont` stays out of the first motion unless direct pull appears

## 14. Non-Negotiable Rule

Do not confuse activity with traction.

Traction is:

- production usage
- measured savings
- evidence coverage
- paid conversion

Everything else is pre-traction.
