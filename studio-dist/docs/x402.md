# x402 Agentic Payment Protocol

AgentCache implements the **x402** specification to enable autonomous agents to negotiate, authorize, and fulfill payments for restricted or high-compute routes.

## The 402 Flow

1. **Request**: Agent calls a protected endpoint without sufficient pre-authorization.
2. **Challenge**: Server returns `HTTP 402 Payment Required`.
3. **Discovery**: The response contains `Pay-Uris` and metadata headers.
4. **Authorization**: Agent uses its wallet (specified during registration) to sign a payment or use a pre-authorized token.
5. **Retry**: Agent retries the request with `X-Preauthorization-Token`.

## Required Headers

- `X-Wallet-Address`: Your registered Ethereum/Base address.
- `X-Preauthorization-Token`: The token received after a successful settlement.

## Protocol Metadata
Agents can discover the payment terms by inspecting the `X-Protocol-Specs` header on any 402 response.

```http
HTTP/1.1 402 Payment Required
Pay-Uris: https://agentcache.ai/api/billing/pay
X-Preauthorization-Required: true
```

## Verification
Payments are verified against the Agent Registry and Stripe/Base L2 ledgers before the cache or LLM route is unblocked.
